1) O que é semântica?

Na programação, a Semântica se refere ao significado de um trecho de código — por exemplo, "que efeito tem a execução dessa linha de JavaScript?", Ou "que finalidade ou função esse elemento HTML tem" (em vez de "como ele se parece?").

2) O que é semântica HTML (semântica no contexto da marcação HTML)?

Em HTML, por exemplo, o h1 (en-US) é um elemento semântico, que fornece o texto que envolve a representação (ou o significado) de "um cabeçalho de nível superior em sua página".

O HTML5 semântico aborda essa deficiência definindo tags específicas para indicar claramente qual o papel desempenhado pelo conteúdo que essas tags contêm. Essas informações explícitas ajudam robôs/rastreadores como Google e Bing a entender melhor qual conteúdo é importante, qual é uma subsidiária, qual é para navegação e assim por diante.

Ao adicionar tags HTML semânticas às suas páginas, você fornece informações adicionais que ajudam o Google e o Bing a entender as funções e a importância relativa das diferentes partes de sua página.

3) O que são elementos semânticos HTML? Dê exemplos de elementos semânticos e não semânticos.

Elementos como 'header', 'footer' e 'article' são todos considerados semânticos porque descrevem com precisão a finalidade do elemento e o tipo de conteúdo que está dentro deles.

“não-semântica” como "div". Eles geralmente davam a esses elementos um atributo classou "id atributo" para descrever seu propósito. Por exemplo, em vez "header" disso, muitas vezes foi escrito como "div class="header""".

4) Cite alguns dos benefícios de se utilizar elementos semânticos (marcação semântica).

Primeiro, é muito mais fácil de ler . Esta é provavelmente a primeira coisa que você notará ao olhar para o primeiro bloco de código usando elementos semânticos. Este é um pequeno exemplo, mas como programador você pode ler centenas ou milhares de linhas de código. Quanto mais fácil for ler e entender esse código, mais fácil será o seu trabalho.
